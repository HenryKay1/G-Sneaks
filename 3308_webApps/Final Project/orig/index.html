Our database is designed to store user and sneaker, then keep track of sales transactions involving between users and the sneakers.User information is added when they sign up for the service and sneaker information is added at our discretion.( hypothetically when we produce the sneakers, or buy from somewhere else.Now, a user ( Sales man/woman) triggers a transaction when they purchase or or many products. Then that information is stored in our database as well. Below list below consists of detailed information of out 3 tables respectively.

    User Table
  • User Id (Primary Key)
  • Email
  • Password
  • Membership Fee
  • Birthday
  • Sneaker Id (Foreign Key)
    Product Table
  • Sneaker Id (Primary Key)
  • Brand
  • Name
  • Stock
  • WholeSale name
  • Retail name
    Transaction Table
  • Transaction Id (Primary Key)
  • User Id (Foreign Key)
  • Product Id (Foreign Key)
  • Quantity
  • Date/Time
  • Tracking Number
